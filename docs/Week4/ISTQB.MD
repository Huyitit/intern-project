### 1.1. What is testing

#### K1: Identify typical test objectives

+ Evaluate work products
+ Causing errors, finding defects
+ Ensuring required coverage of a test object
+ Verifying whether requirements have been fulfilled
+ **Ensure the product work as well as expectation of skateholder**

#### K2: Differentiate testing from debugging

| Testing | Debugging |
| --- | --- |
| wake falures up | find causes of failures |
| directly find defects of the test object | reproduct failure, diagnosis, fix defect |

### 1.2. Why is testing necessary

+ Errors are created by human
+ Defects are produced by errors
+ Defects can be results in failures

### 1.3. Testing principles

#### K2: Explain the seven testing principles

1. Testing shows the presence, not the absenece of defects.
2. Exhaustive testing is impossible.
3. Early testing saves time and money.
4. Defects cluster together.
5. Tests wear out.
6. Testing is context dependent.
7. Absence-of-defects fallacy.
